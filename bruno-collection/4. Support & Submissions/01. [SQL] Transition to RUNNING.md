# 🔧 SQL: Transition Hackathon to RUNNING Stage

**⚠️ ВАЖНО:** Support и Submissions доступны только на стадии RUNNING.

## Подключение к PostgreSQL:

```bash
docker exec -it hackathon-platform-postgres psql -U hackathon_user -d hackathon_db
```

## SQL запрос для перевода в RUNNING:

```sql
UPDATE hackathon.hackathons 
SET starts_at = NOW() - INTERVAL '1 hour',
    stage = 'running'
WHERE id = '<HACKATHON_ID>';
```

**Что подставить:**
- `<HACKATHON_ID>` - замени на значение переменной `hackathonId` из environment Demo

## Пример:

```sql
UPDATE hackathon.hackathons 
SET starts_at = NOW() - INTERVAL '1 hour',
    stage = 'running'
WHERE id = 'a1b2c3d4-e5f6-7890-abcd-ef1234567890';
```

## Проверка:

```sql
SELECT id, name, stage, starts_at, ends_at
FROM hackathon.hackathons 
WHERE id = '<HACKATHON_ID>';
```

Должно вернуть:
- `stage` = `running`
- `starts_at` = час назад

## Выход из psql:

```
\q
```

---

## 📝 Что изменилось:

- **REGISTRATION** → **RUNNING**
- Теперь доступны:
  - ✅ Support System (messages, tickets)
  - ✅ Submissions (create, upload files)
- Недоступны:
  - ❌ Team creation/management
  - ❌ Registration changes
