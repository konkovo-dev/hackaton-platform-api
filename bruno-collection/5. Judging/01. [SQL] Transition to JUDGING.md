# 🔧 SQL: Transition Hackathon to JUDGING Stage

**⚠️ ВАЖНО:** Judging доступен только на стадии JUDGING.

## Подключение к PostgreSQL:

```bash
docker exec -it hackathon-platform-postgres psql -U hackathon_user -d hackathon_db
```

## SQL запрос для перевода в JUDGING:

```sql
UPDATE hackathon.hackathons 
SET ends_at = NOW() - INTERVAL '1 hour',
    stage = 'judging'
WHERE id = '<HACKATHON_ID>';
```

**Что подставить:**
- `<HACKATHON_ID>` - замени на значение переменной `hackathonId` из environment Demo

## Пример:

```sql
UPDATE hackathon.hackathons 
SET ends_at = NOW() - INTERVAL '1 hour',
    stage = 'judging'
WHERE id = 'c140d610-8338-4bfe-8059-e30bd18ba639';
```

## Проверка:

```sql
SELECT id, name, stage, ends_at
FROM hackathon.hackathons 
WHERE id = '<HACKATHON_ID>';
```

Должно вернуть:
- `stage` = `judging`
- `ends_at` = час назад

## Выход из psql:

```
\q
```

---

## 📝 Что изменилось:

- **RUNNING** → **JUDGING**
- Теперь доступны:
  - ✅ Assign submissions to judges
  - ✅ Submit evaluations
  - ✅ View leaderboard (staff only)
- Недоступны:
  - ❌ Create/update submissions
  - ❌ Select final submission
  - ❌ Team management
